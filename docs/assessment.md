## Scenario: Charlton Kings Tree Guardian Project

Through this project, we (Charlton Kings Tree Guardian) would like to inspire and motivate people of all ages in our community to become more curious, interested, and active in caring for trees on their street and public spaces in Charlton Kings.

We want interactions to seed further engagement. For example, kids being able to tag a tree near them (school, home) as "Jesse found this tree on 09.04.26" and encouraging them to "find a friend to adopt a local tree".

It would be great to allow people to upload relevant photos to show the tree is healthy or has bird / insect life around it or biodiversity, e.g., "beautiful lichen".

It needs to be super easy to use and optimised for mobile phones so it's easy to interact with when we run community events and festivals and want to log information live.

Other ideas are a Tree Guardian Loyalty Scheme, where they can earn points for checking the trees health and following local arborists advice on how to care for their tree when it's exceptionally hot or cold, or if they spot a sign of disease. As this requires knowledge of specific tree types and what to look for, we would need to create an FAQ section or link for more info, perhaps in collaboration with Gloucestershire Wildlife Trust.

As a first step, we would like to be able to build a database of the street and public trees in Charlton Kings with information about each specific tree, such as its type, probable age, location, size, when planted and by whom, if "adopted" and by whom, relevant photographs, any text notes, and enable members of the public to add information such as: tagging that they have spotted it; signs of disease; any wildlife spotted in the tree (birds, insects, mammals); etc. We would like to be able to add to and update the database through an easy-to-use mobile interface.

Ideally, we would like to be able to display the trees on a map and attach a QR code to each tree so members of the public can find out information about that tree.

If time permits, we would like to explore the other ideas mentioned above.

Currently, we are using the app iNaturalist to record observations of trees. This holds information about the type of tree, a photograph, and its location as a grid reference. We would like to be able to upload this information onto the database and add other fields as mentioned above.

- Report shows appropriate understanding of Scrum and iterative, incremental development.

  - Collaborated equitably throughout the project across a range of technical, decision making and administrative tasks.
  - Contributed to a good team spirit and helped the team collectively flourish and overcome challenges.

- Code clearly documented.

### To achieve 60+

As 50+ with:
- Personal contribution shows how you learned from mistakes and became an asset to future development projects.

  - Different types of Forms and Report pages
  - Different types of page input items such as Check Boxes, Radio Buttons, Dropdown List etc
  - Navigation - Breadcrumbs, Tabs, Navigation Bars etc
  - Helpful messages for the users

### To achieve 70+

As 60+ with:
- A very good application of Scrum and agile project-management throughout the project; good team evolution and effective demonstration of methods to ensure team cohesion and focus; extended consideration of business functionality, business processes and system design.
- Excellent report; all sections written to a high standard and very clearly structured.
